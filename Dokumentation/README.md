# Dokumentation – Labor Echtzeitsysteme

Inhaltsverzeichnis aller Aufgaben. Jede Aufgabe wird über einen eigenen Pull Request abgegeben.

## Dateistruktur pro Aufgabe

Jede Aufgabe besteht aus **zwei getrennten Dateien**:

| Datei | Inhalt | Wer bearbeitet? |
|-------|--------|-----------------|
| `Aufgabe_XX.md` | Aufgabenstellung, Anforderungen, Quellen | ⛔ **Nicht verändern** – wird vom Dozenten gepflegt |
| `Bearbeitung_XX.md` | Eure Dokumentation, Antworten, Code-Snippets | ✅ **Hier arbeiten** – eure Abgabe |

> ⚠️ **Wichtig:** Ändert **niemals** die `Aufgabe_XX.md`-Dateien. Wenn der Dozent Aktualisierungen über einen Merge einspielt, bleibt eure Arbeit in `Bearbeitung_XX.md` konfliktfrei.
